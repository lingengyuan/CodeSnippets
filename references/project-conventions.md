# CodeSnippets — 归档规范

> 本文件供 insight_agent.py 和 Claude Code 的 insight-collector skill 读取。
> 所有归档操作必须遵循以下规范。

---

## 目录路由

| 内容类型 | 目标目录 | 示例文件名 |
|---------|---------|---------|
| Python 代码 | `python/` | `fts5_fuzzy_search.py` |
| JavaScript / Node.js 代码 | `javascript/` | `pdf_to_images.html` |
| Rust / Go / 其他语言代码 | `snippets/` | `kway-merge-heap.rs` |
| 跨语言通用片段 | `snippets/` | `atomic-file-write.rs` |
| 独立单页 HTML 工具 | `html-tools/` | `pdf_ocr.html` |
| 项目/配置模板（复制即用） | `templates/` | `WORKFLOW.md.template` |
| 项目灵感与构思 | `ideas/` | `yt-browse-local-first.md` |
| 外部资料精读与参考 | `analysis/` | `simon-willison-agentic-patterns.md` |

**文件命名规则**：小写字母 + 连字符，描述性，不含空格。

## Analysis 精读文档格式（`analysis/{topic}.md`）

```markdown
# {标题} 精读

**来源**: [{标题}]({url})
**日期**: YYYY-MM-DD
**标签**: tag1, tag2, tag3

---

## 30秒 TL;DR

> {一段话核心论点——读完就知道这篇文章解决了什么问题}

## 概念总览

| 概念/模式 | 核心思想 | 适用场景 |
|---------|---------|---------|

---

## 深读

{从原文各章节展开，逐点分析}

---

## 心智模型

> {作者看问题的框架}

**适用条件**：{...}
**失效条件**：{...}

---

## 非显见洞见

- **洞见**：{具体结论}
  - 所以：{推论}
  - 因此可以：{行动}

---

## 反模式与陷阱

- **{陷阱名}**：{描述} → 正确做法：{...}

---

## 与现有知识库的连接

- 关联 `path/to/file`：{关联说明}

---

## 衍生项目想法

### {想法标题}

**来源组合**：[本次 X] + [已有 Y]
**为什么有意思**：{非显见的价值}
**最小 spike**：{1 天内能完成的实验}
```

## 与现有知识库的连接

- 关联 `path/to/file`：{...}

## 下一步行动

- [ ] {最小可行 spike}

## README.md 目录表更新格式

在 **英文表** 和 **中文表** 各插入一行：

```
| {emoji} | `path/to/file` | {功能描述（英文）} | {依赖} |
| {emoji} | `path/to/file` | {功能描述（中文）} | {依赖} |
```

**类型 emoji**：
- 🟢 代码——可直接运行或导入
- 🟡 模板——复制后按需修改
- 🔵 参考——做设计决策时查阅

---

## READING_LIST.md 同步规则

归档完成后，更新 `/root/projects/CodeSnippets/READING_LIST.md`：
- 若来源 URL 在"待归档"有 `- [ ]` 条目 → 改为 `- [x]`
- 若 URL 不在列表 → 在"已归档"末尾追加 `- [x] {url}`

---

## 质量原则

1. **不虚构内容**——只提取原材料中实际存在的信息
2. **代码用英文，文档用中文**
3. **一个来源可产出多个文件**（如：博文 → 代码片段 + idea 文档）
4. **URL 来源必须包含 `来源` 字段**
5. **衍生想法必须注明来源组合和最小 spike**——没有这两项的想法是无效的
